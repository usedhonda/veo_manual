# Veo 3.1最適化出力ガイド

## 概要

Nano Banana Proで生成した画像をVeo 3.1で最適に使用するための技術的なガイドです。

**目的**:
- 画像→動画の品質最大化
- 技術的な互換性確保
- 予期せぬ問題の回避

---

## 解像度とアスペクト比

### Veo 3.1の対応形式

| アスペクト比 | 用途 | 入力推奨解像度 |
|-------------|------|---------------|
| 16:9 | 横長動画（標準） | 1920x1080以上 |
| 9:16 | 縦長動画（TikTok等） | 1080x1920以上 |

### 非対応形式

```
【非対応】
- 1:1 （正方形）
- 4:3
- 21:9 （ウルトラワイド）
- その他のカスタム比率

【対処法】
必要に応じてクロップまたは余白追加で
16:9または9:16に調整
```

### 解像度の考慮

```
【入力画像】
- 推奨: 1920x1080以上
- Veoが内部で適切にリサイズ
- 高解像度ほど品質向上の可能性

【出力動画】
- 標準モデル: 1080p
- Fastモデル: 720p
- Video Extension: 720p固定
```

## 画像品質の最適化

### 必須条件

```
【品質要件】
- シャープでクリア
- ノイズなし
- 圧縮アーティファクトなし
- 適切な露出

【避けるべき問題】
- ブラー（ぼけ）
- モアレ
- 色かぶり
- 過度なコントラスト
```

### ファイル形式

```
【推奨】
- PNG（ロスレス、最高品質）
- JPEG（高品質設定、90%以上）

【注意】
- 透過PNGは背景が追加される可能性
- GIFは非推奨
- RAW/TIFFは要変換
```

## 構図の最適化

### 動きを考慮した余白

```
┌─────────────────────────────────────────────────────────────┐
│ カメラワーク別の余白設計                                    │
│                                                             │
│ 【オービット（周回）】                                      │
│  ┌─────────────────┐                                       │
│  │    ○○○○○       │  被写体の周囲に均等な空間             │
│  │   ○ 製品 ○      │  最低でも各辺20%の余白               │
│  │    ○○○○○       │                                       │
│  └─────────────────┘                                       │
│                                                             │
│ 【ドリーイン（前進）】                                      │
│  ┌─────────────────┐                                       │
│  │                 │  前方に広い空間                       │
│  │     [被写体]    │  被写体は奥側1/3に配置                │
│  │   ←  前進方向  │                                       │
│  └─────────────────┘                                       │
│                                                             │
│ 【パン（横移動）】                                          │
│  ┌─────────────────┐                                       │
│  │ ←─────────────→│  横方向に広い構図                     │
│  │   [  広い風景  ] │  移動方向に十分な空間                 │
│  └─────────────────┘                                       │
└─────────────────────────────────────────────────────────────┘
```

### 被写体の配置

```
【三分割法】
- 重要な要素を交点に
- 動きの方向に空間を残す
- バランスの取れた構図

【中央配置】
- オービットに最適
- 安定感のある構図
- 製品撮影に適する
```

## First Frame最適化

### Image-to-Videoでの使用

```
【最適化ポイント】
1. 動きの開始点として自然な状態
2. ライティングの方向を明確に
3. 被写体の姿勢・位置を計算

【例: ドリーイン】
- 被写体をやや遠くに
- 焦点が当たるポイントを意識
- 前景に空間または要素
```

### 避けるべき構図

```
【問題のある構図】
- 被写体が端すぎる（動きで切れる）
- 余白がない（動きの余地なし）
- 複雑すぎる構図（動きで混乱）

【改善方法】
- 適度な余白を確保
- シンプルな構成
- 動きの軌道を予測
```

## Last Frame最適化

### Frame Interpolationでの使用

```
【最適化ポイント】
1. First Frameと一貫したスタイル
2. 変化後の状態を明確に表現
3. 同じライティング条件

【重要】
First FrameとLast Frameは同じ：
- 解像度
- アスペクト比
- 色調
- ライティング方向
```

### シームレスループの作成

```
【方法】
- First Frame = Last Frameに同じ画像
- または位相が繋がる2枚の画像
- 自然なループポイントを選択

【例】
- 炎の揺らぎ（同じフレーム）
- 歩行サイクル（同じポーズ）
- 回転（360度で同じ状態）
```

## 参照画像（Ingredients）最適化

### 要件

```
【技術要件】
- referenceType: "asset" のみ（3.1制限）
- 最大3枚まで
- "style"タイプは非対応

【品質要件】
- クリアで見やすい
- 背景はシンプル
- 特徴が明確に見える
```

### キャラクターシートの仕様

```
【推奨構成】
- 複数アングル（正面、45度、横）
- 表情バリエーション
- 全身または上半身が見える

【避けるべき】
- 複雑な背景
- 他のキャラクターとの混在
- 小さすぎる表示
```

## 色とライティングの整合性

### 色空間

```
【推奨】
- sRGB色空間
- 標準的なガンマ

【注意】
- Adobe RGBからの変換に注意
- HDR画像は要変換
- 極端な彩度は避ける
```

### ライティングの一貫性

```
【動画との整合】
- 影の方向を固定
- 光源の位置を明確に
- 色温度を統一

【時間帯設定】
- dawn: 青みがかった薄明
- golden_hour: 暖かいオレンジ
- noon: 白っぽい強い光
- blue_hour: 冷たい青
- night: 人工光源
```

## トラブルシューティング

### 問題: 動画でスタイルが変わる

```
【原因】
- 画像のスタイルが曖昧
- プロンプトとの不一致

【対策】
- 画像とプロンプトのスタイルを一致させる
- 明確なスタイルキーワードを使用
- ネガティブで他スタイルを除外
```

### 問題: 色が変わる

```
【原因】
- ライティング指定との矛盾
- 色温度の不一致

【対策】
- 画像の色温度に合わせた指定
- 具体的な色を再指定
- ネガティブに「color shift」追加
```

### 問題: 構図が崩れる

```
【原因】
- 余白不足
- 動きとの不整合

【対策】
- 十分な余白を確保
- 動きのシミュレーション
- 控えめな動きから試す
```
