# Nano Banana Pro 基礎

## 概要

Nano Banana Pro（Gemini 2.5 Flash Image）は、Veo 3.1用の入力画像を生成するための静止画生成ツールです。

**役割**:
- Veo 3.1のFirst Frame / Last Frame用画像
- Ingredientsモードの参照画像（キャラクターシート）
- シーン構成の事前確認

---

## なぜNano Banana Proを使うのか

### Veo 3.1との相性

```
┌─────────────────────────────────────────────────────────────┐
│ Nano Banana Pro → Veo 3.1 パイプライン                      │
│                                                             │
│  1. 高品質な開始フレームを生成                              │
│  2. 一貫したスタイルで動画化                                │
│  3. 予測可能な結果を得る                                    │
└─────────────────────────────────────────────────────────────┘
```

### 主な利点

| 利点 | 説明 |
|------|------|
| 構図のコントロール | 動画の開始点を正確に指定 |
| スタイルの一貫性 | 同じ設定で複数画像を生成 |
| 反復の効率化 | 静止画で確認→OK→動画化 |
| キャラの一貫性 | キャラシートで特徴を固定 |

## 基本的なプロンプト構造

### シンプルな依頼

```
Veo 3.1のFirst Frame用に画像を生成してください。

【被写体】
- 何を撮るか

【構図】
- アングル、フレーミング

【背景】
- 場所、環境

【ライティング】
- 光の質、方向

【解像度】
- 16:9, 1080p相当
```

### 詳細な依頼（JSON出力用）

```
Nano Banana Pro用の画像プロンプトを生成してください。

【被写体】
- タイプ: [人物 / 製品 / 風景]
- 詳細: [具体的な説明]
- 位置: [フレーム内の位置]

【構図】
- ショットタイプ: [wide / medium / close-up]
- アングル: [eye_level / low_angle / high_angle]
- 余白: [動画での動きに必要な空間]

【スタイル】
- メディウム: [realistic / anime / stylized]
- 色調: [warm / cool / neutral]
- 雰囲気: [cinematic / commercial / artistic]

【技術仕様】
- 解像度: 1080p
- アスペクト比: 16:9
- 用途: [First Frame / Last Frame / Reference]
```

## 用途別の使い分け

### 1. First Frame用

動画の開始点となる画像。

```
【重要ポイント】
- 動きに必要な余白を確保
- ライティングの方向を意識
- 被写体の位置を計算
```

**例: ドリーイン用**
```
被写体を中央やや奥に配置
前景に空間を確保
カメラが前進する余地を残す
```

### 2. Last Frame用

Frame Interpolationモードで使用。

```
【重要ポイント】
- First Frameと一貫したスタイル
- 変化後の状態を明確に
- 同じライティング条件
```

### 3. 参照画像用（Ingredients）

キャラクターの一貫性を保つための参照。

```
【重要ポイント】
- 複数アングルを含む
- 背景はシンプルに
- 特徴的な要素を明確に
- 色の正確な再現
```

## 解像度とアスペクト比

### Veo 3.1との互換性

| アスペクト比 | 用途 | 推奨解像度 |
|-------------|------|-----------|
| 16:9 | 横長動画（標準） | 1920x1080 |
| 9:16 | 縦長動画（TikTok等） | 1080x1920 |

### 注意点

- Veo 3.1は16:9と9:16のみ対応
- 1:1（正方形）は未対応
- 解像度は自動調整されるが、高解像度推奨

---

## 動画用に最適化するコツ

### 1. 動きの余白を確保

```
【オービット用】
- 被写体の周囲に空間
- 360度どこから見ても良い構図

【ドリーイン用】
- 前方に空間
- 被写体はやや遠く

【パン用】
- 横方向に広い構図
- 被写体は端に寄せる
```

### 2. ライティングの一貫性

```
【動画との整合】
- 影の方向を明確に
- 光源の位置を固定
- 時間帯を指定
```

### 3. 背景の考慮

```
【静的背景】
- 変化しない要素は詳細に

【動的背景】
- 動く要素（雲、水）は指定
- ループの可能性を考慮
```
